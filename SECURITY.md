# Security Policy

## Supported Versions
Currently, we support the latest branch of the repository for security updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_macro:|
| `< 1.0` | :x:                |

## Reporting a Vulnerability

We take the security of our infrastructure and code seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them privately by emailing the repository maintainers. You will find contact information on our GitHub profiles, or you can use GitHub's "Report a Vulnerability" feature if enabled on the repository.

Please include the following information in your report:
- Type of issue (e.g., buffer overflow, SQL injection, hardcoded credentials)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

We will respond to your report within 3 business days and keep you informed of our progress towards a fix and announcement.
